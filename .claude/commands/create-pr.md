# Create PR

현재 브랜치의 커밋을 분석하여 프로젝트 PR 템플릿에 맞는 Pull Request를 생성합니다.

## 작업 절차

1. 다음 정보를 수집한다:
   - `git log develop...HEAD --oneline` — 현재 브랜치의 커밋 목록
   - `git diff develop...HEAD --stat` — 변경된 파일 목록
   - `git diff develop...HEAD` — 상세 변경 내용
   - `gh pr list --state open` — 이미 열린 PR 확인 (중복 방지)

2. 수집한 정보를 바탕으로 PR 본문 초안을 작성한다.
   - **PR 제목**: `<emoji> [<type>] <한국어 설명>` 형식 (커밋 메시지 규칙과 동일)
   - **본문**: 아래 템플릿을 **정확히** 따른다 (섹션 순서, 마크다운 형식 포함)

3. 초안을 사용자에게 보여주고 승인을 받은 뒤 `gh pr create`로 생성한다.

4. **Co-authored-by 줄은 절대 포함하지 않는다.**

## PR 본문 템플릿

아래 템플릿을 그대로 사용하되, 각 섹션을 커밋 내용 기반으로 채운다.

```
## 🛠️ 변경 사항

> 실제로 어떤 작업을 했는지 구체적으로 작성해주세요.

- [ ] UI 수정 (Design)
- [ ] 기능 추가 (Feature)
- [ ] 버그 수정 (Bug)
- [ ] 리팩토링 (Refactor)
- [ ] 성능 개선 (Performance)
- [ ] 테스트 추가 (Chore)
- [ ] 기타:

### 세부 변경 내용

-
-
- ***

## 🔍 관련 이슈

> 관련 이슈를 링크해주세요. ex) close #23, related #23

- #

---

## 📸 스크린샷 / GIF (선택)

> UI 변경이 있다면 첨부해주세요.

| Before | After |
| ------ | ----- |
|        |       |

---

## ⚠️ 주의 사항 / 리뷰 포인트

> 리뷰어가 특히 봐줬으면 하는 부분이나 고민했던 지점을 작성해주세요.

-
- ***

## 🔄 연관 작업

> 후속 작업이나 연관된 PR이 있다면 링크해주세요.

-
-
```

## 채우기 규칙

- **변경 사항 체크박스**: 커밋 타입(feat/fix/refactor/style/perf/test/chore)에 맞는 항목에 `[x]` 표시
- **세부 변경 내용**: 커밋별로 핵심 변경 내용을 bullet으로 요약 (한국어)
- **관련 이슈**: 브랜치명에 이슈 번호가 있으면 자동 추출 (예: `feature/184` → `- close #184`), 없으면 `- #` 그대로 유지
- **스크린샷**: UI 변경(feat/style)이 있으면 표 유지, 없으면 섹션 전체 제거하지 말고 표만 비워둠
- **주의 사항 / 리뷰 포인트**: 복잡한 로직 변경이나 리뷰어가 주의해야 할 부분 작성, 없으면 `-` 유지
- **연관 작업**: 연관 PR이 없으면 `-` 유지

## 출력 형식 (계획 단계)

PR 생성 전에 아래 형식으로 초안을 출력한다:

```
[PR 제목] ✨ [feat] 일정 생성/편집 폼 아코디언 UI 구현

[본문 미리보기]
(전체 본문 내용)
```

승인 후 `gh pr create --base develop --title "..." --body "..."` 로 생성한다.
